## 🔧 커스터마이징 가이드

`snake.html` 내부 스크립트에서 아래 상수를 수정하면 됨

```js
// 보드/타일
const COLS = 25;
const ROWS = 25;
const TILE = 24;

// 속도 레벨
const SPEED_LEVELS = [
  { label: '저', value: 6 },
  { label: '중', value: 9 },
  { label: '고', value: 12 }
];

// 점수 N점마다 속도 +1
const SCORE_SPEED_STEP = 4;
```

### 자주 하는 변경 예시

*   **보드 크기 변경**: `COLS`, `ROWS` 값 조정
*   **고정 속도**로 유지하고 싶으면:
    1.  `recalcSpeed()`에서 점수 기반 가속 제거
    2.  `speed = baseSpeed;` 고정
*   \*\*벽 통과(워프)\*\*로 바꾸려면:
    *   벽 충돌 체크 대신 위치를 모듈로 연산으로 래핑 처리
*   **WASD 지원** 추가:
    *   `keydown` 핸들러에 `w/a/s/d` 분기 추가
*   **모바일 조작**:
    *   화면에 가상 패드 버튼 추가 또는 스와이프 이벤트 처리

## 🧪 동작 원리 개요

*   **게임 루프**: `requestAnimationFrame` + 누적 시간(`acc`)로 일정 스텝 주기 처리함
*   **이동/충돌**: 타일 단위로 머리 좌표 갱신 → 벽/자기 몸 체크 → 먹이 시 스네이크 길이 증가
*   **렌더링**: `<canvas>`에 스네이크/먹이/테두리, 배너(일시정지/게임오버) 표시함
*   **저장**: `localStorage`에 최고점(`snake-high`) 키로 저장함

## ❓ FAQ

**Q. 실행했는데 키가 안 먹음**  
A. 브라우저 포커스가 캔버스/문서에 있는지 확인. 페이지 내 다른 입력 요소가 포커스를 가져가면 입력이 차단될 수 있음

**Q. 최고점이 초기화됨**  
A. 브라우저 캐시/스토리지 삭제, 시크릿 모드 사용, 도메인 변경 시 로컬스토리지 값이 달라질 수 있음

**Q. 화면이 흐릿함**  
A. 고해상도 디스플레이에서 `devicePixelRatio` 반영해 스케일링함. 브라우저 줌(125%, 150% 등)을 기본(100%)으로 맞추면 선명해짐
